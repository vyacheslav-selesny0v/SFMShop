# Git Workflow для проекта SFMShop

## Использованные команды Git
- git init - инициализация репозитория
- git add - добавление файлов в staging area
- git commit -m - сохранение изменений и добаавление комментария
- git push origin <branchname> - отправка изменений в удаленный репозиторий
- git checkout -b <branchname> - создание и переключение на новую ветку
- git log - просмотр истории коммитов
- git log --oneline - просмотр истории коммитов в одну строку
- git log --oneline --graph - просмотр истории коммитов в виде графа
- git branch - просмотр существующих веток
- git pull origin main - стягивание изменений из удаленного репозитория и обновление локального
- git merge - слияние веток
- git fetch - стягивание изменений из удаленного репозитория
- git diff — сравнение и отображение различий между версиями файлов

## Созданные ветки
- main - основная ветка проекта
- feature/add-discount-system - ветка для системы скидок
- feature/add-email-validation - ветка для валидации почты
- feature/add-inventory-management - методы управления складом

## Разрешенные конфликты
- Конфликт в src/models/product.py при слиянии feature/add-discount-system
- Конфликт в src/api/main.py при слиянии feature/test-conflict

## Стратегия работы с ветками
- Основная ветка проекта main
- Для каждой задачи создается отдельная ветка с описательным именем feature/*
- После завершения работы над задачей создается pull request, после ревью и тестов ветка вливается в main
